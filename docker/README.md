# Local backend images

These Dockerfiles (auto-generated by `tools/gen_dockerfiles.py` from each
`scripts/modal/modal_train_*.py`'s `modal.Image` recipe) build one image per backbone for the
**local** execution mode. Each image bakes in the heavy deps **and the model
source code** (PointTransformerV3 / RandLA-Net / KPConvX), so a built image is
fully self-contained.

**Building needs nothing but Docker + Buildx.** Model sources are git-cloned at
pinned SHAs during the build — no local model checkouts and no GPU at build time
(an NVIDIA GPU is still required at RUN time; the images are CUDA-based). Every
other machine just needs the built image — pull it from a registry (or `docker
load` a tarball). At run time the GUI bind-mounts only `/workspace` (this repo),
`/datasets`, and `/outputs`.

## Build (any machine with Docker Buildx)

```bash
bash docker/build_all.sh
```

## Distribute via a container registry (build once, run anywhere)

```bash
# on the build machine — push to e.g. GitHub Container Registry:
docker login ghcr.io
TT_REGISTRY=ghcr.io/<you> bash docker/push_all.sh

# on any other machine — no repos, no build:
TT_REGISTRY=ghcr.io/<you> bash docker/pull_all.sh
```

Then point the GUI at the registry so it pulls instead of expecting a local
build — either set the env var before launching:

```bash
export TT_REGISTRY=ghcr.io/<you>
```

…or set `local_config["registry"] = "ghcr.io/<you>"` in
`~/.config/trainer_gui/state.json`. With a registry configured, `docker run`
auto-pulls a missing image and the GUI's preflight says *"docker will pull it"*
instead of *"build it"*.

> Weights vs. images: a **container registry** (GHCR/Docker Hub) is for the
> images. Use **Hugging Face / S3 / a Modal volume** for your trained `.pth`
> checkpoints so inference is portable too — that's the layer HF is built for.

## Distribute via a tarball (no registry / air-gapped)

```bash
docker save trainer-local-randlanet | gzip > randlanet.tar.gz   # build machine
docker load < randlanet.tar.gz                                  # target machine
```

`build_all.sh` / `push_all.sh` / `pull_all.sh` (and `.ps1` equivalents) are
regenerated by `python tools/gen_dockerfiles.py` — don't hand-edit.
