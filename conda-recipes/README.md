# conda-recipes — model sources & trained weights as conda packages

Everything the pixi training environments need beyond wheels is packaged here
with [rattler-build](https://rattler.build) and hosted on a public
[prefix.dev](https://prefix.dev) channel, so a user needs **zero auth** to
install an environment or a published model. Only maintainers hold an upload
token.

## Package families

| Package | Contents | Platform |
|---|---|---|
| `trainer-src-<key>` | upstream model sources pinned at the SHA the modal recipe clones, installed to `$PREFIX/opt/<pkg>`; `<PKG>_SRC` comes from `envs/pixi.toml` `activation.env` (the packages' activate.d `.sh` scripts are Linux-only and now redundant) | noarch (ptv3, concerto, sonata, utonia) / linux-64 + win-64 with compiled C++ exts (randlanet, kpconv, kpconvx — the keops-free fork removed the pykeops Linux-only dependency) |
| `trainer-weights-<dataset>-<key>` | a finished run's `final_model.pth` + `run.json` under `$PREFIX/share/trainer-weights/<name>/` | noarch (generated by `tools/package_weights.py`, not stored here) |

Upstream pretrained SSL encoders (Concerto/Sonata/Utonia HF checkpoints,
CC-BY-NC) are **not** packaged — they stay runtime-downloaded to
`$TT_OUTPUTS_ROOT/hf_cache/<pkg>`. Don't redistribute them.

## Building & publishing

```bash
pixi run --manifest-path envs/pixi.toml -e pkg build-src          # all recipes -> conda-recipes/output/
pixi auth login prefix.dev --token pfx_...                        # once, maintainers only
TT_PREFIX_CHANNEL=<channel> pixi run --manifest-path envs/pixi.toml -e pkg upload
```

`build-src` builds whatever the host platform supports: on Linux, everything;
on Windows, the three compiled recipes (randlanet, kpconv, kpconvx — noarch
artifacts are built on Linux but install everywhere). Windows builds need
Visual Studio Build Tools (C++ workload) installed.

Then flip the manifest live: in `envs/pixi.toml` uncomment the
`https://repo.prefix.dev/<channel>` channel and each feature's
`trainer-src-<key> = "*"` dep, and `pixi lock --manifest-path envs/pixi.toml`.

## Packaging trained weights

```bash
pixi run --manifest-path envs/pixi.toml -e pkg package-weights /path/to/outputs/runs/<run_id>
pixi run --manifest-path envs/pixi.toml -e pkg upload
```

`package_weights.py` reads the run's `run.json` (backbone, dataset), names the
package `trainer-weights-<dataset>-<key>`, and versions it by date. Users then:

```bash
pixi add --manifest-path envs/pixi.toml --feature <key> trainer-weights-<dataset>-<key>
```

and the Infer page lists the installed weights automatically.

## Updating a pinned model SHA

Bump `context.sha` AND `package.version` in the recipe, mirror the SHA in
`scripts/modal/modal_train_<key>.py`, rebuild + upload. The smoke test's
`tools/check_env_sync.py` fails if the recipe SHA and the modal recipe drift.

## First run on the Linux GPU box (one-time validation checklist)

1. `git pull` + install pixi; `pixi run -e pkg build-src` then `upload`
   (maintainer token). Flip the manifest live (channel + `trainer-src-*` deps,
   see above) and commit the re-lock.
   **randlanet: rebuild + re-upload is REQUIRED** — the published linux-64
   build 1 regenerated knn.cpp under the module name `knn`, so its
   `nearest_neighbors` extension has the wrong `PyInit_*` symbol and fails at
   import. Build ≥2 (this recipe) fixes it.
   **kpconvx: rebuild + re-upload is REQUIRED too** — published linux-64 1.0.0
   is the old pykeops sources; the keops-free fork line is 1.1.0.
2. Per backbone: GUI **Configure model… → Install / update env** (or
   `pixi install -e <env>`), then `pixi run -e <env> sanity` — proves wheels,
   compiled exts, and the `<PKG>_SRC` activation var. If the activation var
   doesn't survive `pixi run`, switch the recipes' activate.d export to a
   `[feature.<key>.activation.env]` entry in envs/pixi.toml.
3. Smoke-train 2 epochs per backbone (GUI smoke checkbox); one local inference
   incl. an ensemble member; kpconvx run exercises the torch-cluster neighbor
   backend (keops-free fork);
   concerto/sonata/utonia pull their HF encoders to
   `$TT_OUTPUTS_ROOT/hf_cache/`; one run with an out-of-workspace dataset
   (TT_DATASET_DIR); DataLoader num_workers>0 across an epoch boundary.
4. The goal round-trip: `pixi run -e pkg package-weights <run_dir>` → `upload`
   → on a clean checkout, install the env + `pixi add` the weights package →
   Infer page **Installed…** → run inference.
5. Only after all of that: prune the old ghcr.io trainer-local-* images.
